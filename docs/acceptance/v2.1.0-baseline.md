# v2.1.0 acceptance baseline

Baseline target: the v2.0.9 release candidate at `c1e9f8172f32957b74146e8c0b3189ea45d24c7f`.
Published overlay compatibility is also checked from v2.0.8.

## Required contracts

1. Project/task/lane/operation identities are opaque and exact; linked
   worktrees share project identity, while task/model/domain lanes do not.
2. Concurrent or sequential review starts cannot overwrite another operation.
   Initial and verify share one surface; a later cycle resumes the same exact
   task/model/domain or reports a visible fresh fallback.
3. Reviewer, secure-fetch, and secure-synth permission boundaries are reapplied
   on every start/resume. Stored shell commands are never executed.
4. New cmux splits are anchored to the caller UUID and opened to its right.
   Missing anchor/resume capability is visible and never falls back to another
   vertical tab/workspace.
5. Lane work is FIFO and idempotent. Crash/surface disappearance releases the
   lane and permits queued work to continue. Reap is serialized with enqueue.
6. Reap archives all exact review cycles, clears runtime homes only after task
   archive, and never treats review approval as task completion.
7. Overlay preflight blocks every active legacy or broker session. v1/v2 data
   remains readable but no running legacy process is adopted.

## Verification gate

- `make test`, schema/contract/model-literal lints, DCG smoke, task-session
  concurrency/fault tests, review-dispatch regression, and research isolation.
- Dynamic installed-skill matrix for Claude and Codex, plus focused live
  anchored-spawn, checkpoint/resume, read-only review, vaultless fetch, and
  networkless synth scenarios.
- macOS fresh install, published-v2.0.8 overlay, and local-v2.0.9-RC overlay.
- One full Fable/high review of `main...HEAD`; fixes are verified in that same
  reviewer session before the branch is release-ready.

Automatic archive garbage collection and Windows support remain explicit
feature gaps.

## Repo-shipped live runner

`make acceptance-live` executes the dynamic skill/runtime matrix through
`scripts/live-acceptance-runner.py`; no external `EVAL_RUNNER` is required.
Every cell opens an interactive Claude/Codex surface to the right of the exact
caller, checks out the committed source HEAD into a disposable local clone,
loads the named skill from that clone, executes its exact registry-owned live
fixture, and requires a bounded typed outbox with model, effort, actual
behavior, cleanup, and evidence. Conversation fixtures define their observable
single-turn response as completion, so `clarify`, quiz, and draft cells never
pause for an unattended human reply. The runner closes only the captured
surface UUID and removes the clone only after the agent exits.
Missing cmux, authentication, dependencies, output, cleanup, or identity parity
is a visible `blocked`/`fail` cell rather than a synthetic pass.

Scenario classes live in `evals/acceptance/scenarios.json`; exact per-skill
fixtures live beside expectations in `evals/acceptance/skills.json`.
`acceptance-check` fails when an installed skill has no bounded fixture or when
the scenario registry differs from the scenarios referenced by the dynamic
skill inventory. The cell runner also rejects stdin rows whose scenario or
expected result does not match that canonical inventory.
