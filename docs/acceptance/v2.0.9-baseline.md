# v2.0.9 acceptance baseline

Baseline target: published `v2.0.8` (`9161a94034a1648f5a6de5323ab6cb9112a3ceaf`).
The reusable harness was added in a separate commit and points back to this
immutable target; it does not change v2.0.8 product behaviour.

## Deterministic baseline

- Dynamic inventory: 27 skills × Claude/Codex = 54 required live cells.
- `make test`: pass with Homebrew Python 3.14.4.
- Generated Codex marketplace: current.
- Vault schema/validation: pass.
- Existing lifecycle tests cover task/reviewer exit, callbacks, escalation,
  watchdog, recovery, surface cleanup, runtime hooks, protected research,
  dense degradation, DCG assets, and transactional vault writes.
- Environment finding: a login shell can select an older system Python without
  `tomllib`; v2.0.9 preflight must report the interpreter actually selected by
  hooks and child launchers.

## Contract feature gaps

These are expected v2.0.9 additions, not regressions in the published release:

1. Runtime model/effort defaults are duplicated across supervisors, launchers,
   native profiles, skills, documentation, and exact-string tests.
2. Dispatch does not inherit an explicit current-session routing snapshot.
3. Task/review metadata records effective model values but not normalized
   routing intent, source, or config fingerprint.
4. SessionStart has maintenance nudges but no common, deterministic dependency
   and routing readiness preflight.
5. Protected research pins its Codex model locally and has no explicit
   unsafe-inherit-session route.
6. Hybrid retrieval already degrades safely to sparse, but setup/session
   diagnostics do not give one consolidated Ollama/`bge-m3` repair path.
7. Release acceptance does not yet enforce dynamic per-skill runtime parity.

## Live baseline gate

The checked-in matrix is the required ledger for the live Claude/Codex pass.
Every row must record model, effort, expected/actual behaviour, cleanup,
evidence, defect, and decision. A retry does not erase the first failure;
`blocked`, flaky, and unapproved `n-a` remain non-green.

Raw terminal/model transcripts remain local. Only bounded, content-free or
sanitized evidence is eligible for the version-controlled report.
