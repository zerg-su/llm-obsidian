# Reap-send compatibility

Load only for legacy metadata or explicit interactive diagnosis.

The historical path may render canonical Markdown from `.task-summary.json`,
read the exact coordinator surface and configured reap command, and send one
bounded callback. It must first validate task/session/plan identity, preserve
the final result type/title, and reject duplicates or mismatched summaries.

Do not scrape a summary from screen prose when the canonical JSON exists. Do
not mutate the coordinator vault, send a second `/reap`, close the task/reviewer
surface, delete worktrees, push, deploy, or publish. Unknown/missing surface or
session identity is a visible failure, never a focus-based guess.

Executable truth is `skills/reap-send/scripts/send_reap.py`,
`scripts/reap-send-runner.py`, `scripts/task_contract.py`, and lifecycle tests.
